## Context

Maestro 是一个多 Agent 编排 CLI 工具，采用 Human-to-Agents hub-spoke 模型。项目需要视觉标识和完善的文档来提升项目形象和可访问性。当前项目根目录缺少 README 文件和项目概念图。

## Goals / Non-Goals

**Goals:**
- 使用 canvas-design 技能生成一张概念艺术图，展现"人类指挥家 + 小龙虾乐队"的隐喻
- 创建结构完整的英文 README.md
- 创建对应的中文 README_CN.md
- README 中嵌入项目概念图

**Non-Goals:**
- 不涉及代码逻辑修改
- 不创建 API 文档或详细的开发者指南（后续可独立处理）
- 不设计 Logo 或品牌系统

## Decisions

### 1. 图片生成方式：使用 canvas-design 技能
**选择**: 使用 `canvas-design` skill 生成 PNG 格式概念图
**理由**: 该技能专门用于创建视觉设计作品，支持 PNG 输出，适合生成艺术性质的项目图
**备选**: 手动绘制或外包设计——成本高、周期长

### 2. 文档语言分离：独立文件
**选择**: `README.md`（英文）+ `README_CN.md`（中文），两个独立文件
**理由**: 符合开源项目惯例，便于各语言独立维护，避免单文件过长
**备选**: 单文件双语切换——可读性差，维护成本高

### 3. 概念图构图设计
**选择**: 舞台场景，上方为人类指挥家（代表开发者/Maestro），下方为一群小龙虾（代表 Claw Agent）各持不同乐器演奏
**理由**: 完美对应 Maestro 的 hub-spoke 模型——一个指挥者协调多个执行者
**风格**: 温暖、有趣、略带卡通风格的插画，色调以暖色为主

### 4. README 内容结构
**选择**: 标准开源项目 README 结构
- 项目标题 + 概念图
- 简介（一句话描述）
- 核心特性
- 快速开始（安装 + 基本使用）
- 架构概览
- 贡献指南链接
- 许可证

## Risks / Trade-offs

- [生成图质量] → 可能需要多次迭代 prompt 来调整风格；先生成一版再根据反馈调整
- [中英文内容同步] → 两个文件可能随时间产生差异；初始版本确保内容完全对应
